# 🤖 Discord Bot Hosting Panel

منصة شخصية لاستضافة بوتات الديسكورد مع إمكانية التحكم الكامل وإدارة البوتات.

## ✨ الميزات

- **استضافة عبر التوكن**: أدخل توكن البوت وشغّله فوراً
- **استضافة عبر ZIP**: ارفع كود البوت بالكامل كملف مضغوط
- **لوحة تحكم**: عرض جميع البوتات وحالتهم (شغال/متوقف/معطل)
- **تحكم كامل**: تشغيل، إيقاف، إعادة تشغيل، حذف البوتات
- **سجل الأحداث**: مشاهدة logs البوت بشكل مباشر
- **حفظ البيانات**: كل بوت له مساحة تخزين دائمة خاصة به
- **إعادة تشغيل تلقائي**: يعيد تشغيل البوت في حال تعطل
- **آمن**: حماية بكلمة مرور للوحة التحكم

## 📋 المتطلبات

- Python 3.8+
- pip (مدير حزم Python)
- Node.js (اختياري، لتشغيل بوتات JavaScript)

## 🚀 التثبيت والتشغيل

### 1. تحميل المشروع
```bash
cd ~/discord-bot-hosting
```

### 2. تثبيت المتطلبات
```bash
pip install -r requirements.txt
```

### 3. تشغيل الخادم
```bash
python app.py
```

أو باستخدام سكريبت التشغيل:
```bash
bash start.sh
```

### 4. فتح المتصفح
- الرابط: `http://localhost:5000`
- كلمة المرور الافتراضية: `admin123`

## 🎯 طرق الاستضافة

### الطريقة 1: باستخدام التوكن فقط
1. اذهب إلى `إضافة بوت ← بواسطة التوكن`
2. أدخل اسم البوت
3. أدخل توكن البوت (من [Discord Developer Portal](https://discord.com/developers/applications))
4. اضغط "إضافة البوت"
5. سيتم إنشاء بوت بسيط يقوم بالاتصال بالديسكورد والبقاء متصلاً

### الطريقة 2: باستخدام ملف ZIP
1. جهز مشروع البوت الخاص بك
2. ضغط المشروع في ملف `zip`
3. اذهب إلى `إضافة بوت ← بواسطة ZIP`
4. أدخل اسم البوت والتوكن
5. ارفع ملف ZIP
6. سيتم استخراج الكود وتثبيت المتطلبات وتشغيل البوت

│   ├── add_bot_token.html # إضافة بوت بالتوكن

## 🔒 الأمان

- جميع كلمات المرور مشفرة
- التوكنات مخزنة بشكل آمن في قاعدة البيانات
- كل بوت يعمل في process منفصل
- لا يمكن الوصول للوحة التحكم بدون كلمة مرور

## ⚙️ الإعدادات

### متغيرات البيئة
| المتغير | الوصف | القيمة الافتراضية |
|---------|-------|------------------|
| `ADMIN_PASSWORD` | كلمة مرور لوحة التحكم | `admin123` |
| `SECRET_KEY` | مفتاح تشفير Flask | `discord-bot-hosting-secret-key...` |
| `PORT` | منفذ الخادم | `5000` |
| `DEBUG` | وضع التصحيح | `false` |

### تغيير كلمة المرور
1. اذهب إلى `الإعدادات` في لوحة التحكم
2. أدخل كلمة المرور الجديدة
3. اضغط حفظ

أو عبر متغير البيئة:
```bash
export ADMIN_PASSWORD=my_secure_password
```

## 🛠 التطوير والإضافة

### إضافة بوت بلغة JavaScript/Node.js
1. جهز مشروع JavaScript
2. أضف ملف `main.js` أو `bot.js` كمشغل رئيسي
3. أضف `package.json` مع الاعتمادات
4. ارفع كـ ZIP

### هيكل البوت المرفوع
```
bot-project/
├── main.py          # (أو bot.py, index.py) نقطة الدخول
├── requirements.txt # (اختياري) الاعتمادات
├── data/            # (اختياري) بيانات البوت
└── ...             # ملفات أخرى
```

## 📝 ملاحظات

- البوتات تعمل كعمليات منفصلة (subprocesses)
- في حال تعطل البوت، يتم إعادة تشغيله تلقائياً (إذا كان الخيار مفعّلاً)
- البيانات محفوظة حتى بعد إعادة تشغيل البوت أو انقطاع الاتصال
- منصة Termux قد تحتاج لضبط إضافي للبقاء في الخلفية

## 🐛 المشاكل المعروفة

- **على Termux**: قد يتم إيقاف العمليات إذا أغلقت التطبيق
- **حل**: استخدم `screen` أو `tmux` لتشغيل الخادم في الخلفية

```bash
# تشغيل باستخدام screen
screen -S bot-hosting
python app.py
# Ctrl+A ثم D للخروج من الجلسة
# screen -r bot-hosting للعودة
```
